## 2. 产品背景与目标

### 2.1 行业背景

随着美容行业竞争加剧，客户获取成本不断上升，客户留存和精细化管理成为美容院经营的关键。传统的手工记录方式已无法满足现代美容院的管理需求，数字化、智能化的客户管理系统成为行业发展的必然趋势。

### 2.2 当前痛点

1. **客户信息分散**：客户资料散落在纸质档案、微信、Excel等多个渠道，难以统一管理和查询
2. **预约效率低下**：电话预约易冲突、易遗忘，客户到店等待时间长
3. **客户流失严重**：缺乏有效的客户维系手段，无法及时跟进潜在客户和沉睡客户
4. **库存管理混乱**：产品进销存依靠人工盘点，容易出现缺货或积压
5. **数据决策缺失**：经营数据靠人工统计，时效性差，难以支撑经营决策
6. **项目消耗不清**：客户购买的次卡、储值卡使用情况不透明，容易产生纠纷

### 2.3 产品目标

#### 2.3.1 总体目标

打造一套功能完善、操作便捷、数据安全的美容院客户关系管理系统，帮助美容院实现客户全生命周期管理，提升客户满意度和忠诚度，提高经营效率和盈利能力。

#### 2.3.2 具体目标

| 目标维度 | 具体目标 | 衡量指标 |
|---------|---------|---------|
| 客户管理 | 建立完整的客户档案体系 | 客户信息完整度 ≥ 95% |
| 预约效率 | 实现线上预约和自动排程 | 预约处理时间 ≤ 2分钟 |
| 客户维系 | 自动化客户关怀提醒 | 客户回访率 ≥ 80% |
| 库存管理 | 实时库存监控和预警 | 库存准确率达到 99% |
| 数据分析 | 多维度经营数据可视化 | 报表生成时间 ≤ 5秒 |
| 项目消耗 | 透明化的项目和余额管理 | 客户满意度 ≥ 90% |

### 2.4 目标用户

- **单店美容院**：中小型美容院，客户量500-5000人
- **连锁美容院**：多门店经营，需要统一管理和数据汇总
- **美容工作室**：个人或小型团队经营，注重客户精细化管理

### 4.2 客户信息管理模块

#### 4.2.1 功能概述

客户信息管理是系统的核心模块，用于建立完整的客户档案，实现客户全生命周期管理。

#### 4.2.2 功能清单

| 功能点 | 功能描述 | 优先级 | 复杂度 |
|--------|---------|--------|--------|
| 客户档案 | 记录客户基础信息、联系方式、个人偏好 | P0 | 中 |
| 客户来源 | 标记客户获取渠道，支持来源统计 | P0 | 低 |
| 客户标签 | 自定义标签对客户进行多维分类 | P1 | 低 |
| 客户分类 | 自动/手动划分客户等级（潜在/新/老/VIP） | P1 | 中 |
| 客户查询 | 多条件组合查询和模糊搜索 | P0 | 中 |
| 客户导入导出 | 支持Excel批量导入和导出 | P2 | 中 |

#### 4.2.3 详细需求

##### 4.2.3.1 客户档案管理

**客户基础信息字段**：

| 字段名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| 客户ID | 字符串 | 是 | 系统生成唯一标识 |
| 姓名 | 字符串 | 是 | 客户真实姓名 |
| 手机号 | 字符串 | 是 | 11位手机号，唯一 |
| 性别 | 枚举 | 是 | 男/女/未知 |
| 生日 | 日期 | 否 | 用于生日提醒和关怀 |
| 年龄段 | 枚举 | 否 | 18-25/26-35/36-45/46-55/55+ |
| 微信号 | 字符串 | 否 | 客户微信号 |
| 客户来源 | 枚举 | 是 | 抖音/美团/小红书/微信/朋友介绍/自然进店/其他 |
| 客户状态 | 枚举 | 是 | 潜在客户/新客户/老客户/VIP/沉睡客户 |
| 登记时间 | 日期时间 | 是 | 首次录入系统时间 |
| 备注 | 文本 | 否 | 特殊注意事项 |

**客户扩展信息**：
- 皮肤类型（干性/油性/混合性/敏感性/中性）
- 过敏史
- 职业（选填）
- 家庭住址（选填）
- 消费偏好

**操作需求**：
- 支持新增、编辑、删除客户信息
- 删除客户需确认，且逻辑删除保留数据
- 手机号唯一性校验
- 敏感信息脱敏显示（手机号中间4位*号）

##### 4.2.3.2 客户来源渠道

**来源渠道列表**：

| 渠道名称 | 渠道标识 | 说明 |
|---------|---------|------|
| 抖音 | DOUYIN | 抖音短视频/直播引流 |
| 美团 | MEITUAN | 美团点评平台 |
| 小红书 | XIAOHONGSHU | 小红书种草 |
| 微信 | WECHAT | 微信朋友圈/公众号 |
| 朋友介绍 | REFERRAL | 老客转介绍 |
| 自然进店 | WALKIN | 自然到店 |
| 其他 | OTHER | 其他渠道 |

**功能需求**：
- 客户登记时必选项
- 支持来源渠道数据统计
- 支持按来源筛选客户
- 支持渠道有效性分析

##### 4.2.3.3 客户标签

**系统预设标签**：
- 高消费力
- 价格敏感
- 注重品牌
- 喜欢尝试新品
- 定期护理
- 偶尔护理

**自定义标签**：
- 支持门店自定义标签
- 标签支持颜色标记
- 一个客户可打多个标签
- 支持按标签筛选客户

##### 4.2.3.4 客户分类规则

**自动分类规则**：

| 客户类型 | 分类规则 |
|---------|---------|
| 潜在客户 | 已登记但未消费 |
| 新客户 | 首次消费30天内 |
| 老客户 | 消费超过3次且最近一次消费在90天内 |
| VIP客户 | 累计消费满5000元或单次消费满2000元 |
| 沉睡客户 | 90天未消费的老客户 |

**功能需求**：
- 系统每日自动更新客户分类
- 支持手动调整客户分类
- 分类变化记录日志

#### 4.2.4 界面需求

- 客户列表页：表格展示，支持分页、排序、筛选
- 客户详情页：Tab页展示基础信息、消费记录、项目余额、预约历史
- 客户编辑页：表单编辑，字段校验

### 4.3 预约管理模块

#### 4.3.1 功能概述

实现客户预约的登记、排程、提醒和管理，优化客户到店体验，提高门店运营效率。

#### 4.3.2 功能清单

| 功能点 | 功能描述 | 优先级 | 复杂度 |
|--------|---------|--------|--------|
| 预约登记 | 前台/客户创建预约 | P0 | 中 |
| 预约日历 | 日历视图展示预约排程 | P0 | 高 |
| 预约排程 | 智能推荐可预约时间段 | P1 | 高 |
| 预约状态 | 待确认/已确认/已完成/已取消 | P0 | 中 |
| 预约提醒 | 自动发送提醒通知 | P0 | 中 |
| 预约统计 | 预约率、爽约率统计 | P1 | 低 |

#### 4.3.3 详细需求

##### 4.3.3.1 预约登记

**预约信息字段**：

| 字段名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| 预约ID | 字符串 | 是 | 系统生成 |
| 客户ID | 外键 | 是 | 关联客户 |
| 预约项目 | 多选 | 是 | 可选择多个项目 |
| 预约美容师 | 外键 | 否 | 可指定或系统分配 |
| 预约日期 | 日期 | 是 | 预约到店日期 |
| 预约时间 | 时间 | 是 | 具体时间段 |
| 预约时长 | 整数 | 是 | 预计服务时长（分钟） |
| 预约状态 | 枚举 | 是 | 待确认/已确认/已完成/已取消 |
| 预约人 | 外键 | 是 | 登记人 |
| 备注 | 文本 | 否 | 特殊需求说明 |
| 创建时间 | 日期时间 | 是 | 预约创建时间 |

**预约时间规则**：
- 营业时间为 9:00-21:00
- 时间粒度为30分钟
- 同一美容师同一时间段只能有一个预约
- 可设置每位美容师的最大同时服务人数

**冲突检测**：
- 同一时间段同一美容师已有预约时提示冲突
- 客户同一时间段已有其他预约时提示

##### 4.3.3.2 预约日历

**视图模式**：
- 日视图：展示当天所有预约，按时间线排列
- 周视图：展示一周预约，按美容师分组
- 月视图：展示整月预约概览

**交互需求**：
- 点击时间段快速创建预约
- 拖拽调整预约时间
- 点击预约卡片查看详情
- 不同状态用不同颜色标识

##### 4.3.3.3 预约状态流转

```
┌──────────┐    确认     ┌──────────┐    完成     ┌──────────┐
│  待确认   │ ──────────▶ │  已确认   │ ──────────▶ │  已完成   │
└────┬─────┘             └────┬─────┘             └──────────┘
     │                        │
     │ 取消                   │ 取消
     ▼                        ▼
┌──────────┐               ┌──────────┐
│  已取消   │               │  已取消   │
└──────────┘               └──────────┘
```

**状态说明**：
- 待确认：预约已提交，等待确认
- 已确认：预约确认，客户会到店
- 已完成：客户已到店完成服务
- 已取消：预约被取消

##### 4.3.3.4 预约提醒

**提醒时机**：

| 提醒类型 | 触发时机 | 提醒方式 |
|---------|---------|---------|
| 预约成功提醒 | 预约创建后 | 站内信 |
| 预约确认提醒 | 预约确认后 | 站内信 |
| 到店前提醒 | 预约时间前2小时 | 站内信+短信 |
| 爽约提醒 | 预约时间已过且状态仍为"已确认" | 站内信 |

**提醒内容模板**：
- 支持自定义提醒内容
- 支持变量替换（客户姓名、预约时间、项目名称等）

#### 4.3.4 界面需求

- 预约列表页：表格展示，支持状态筛选
- 预约日历页：多视图日历，支持快速操作
- 预约创建页：表单填写，智能推荐时间
- 预约详情页：展示完整预约信息，支持状态变更
