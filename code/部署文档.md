# BCRM 系统部署文档

| 配置项 | 最低要求 | 推荐配置 |

| CPU | 2核 | 4核+ |

| 软件 | 版本要求 |
|------|---------|
| JDK | 17+ |
| MySQL | 8.0+ |
| Node.js | 18+ (仅构建时需要) |
| Nginx | 1.18+ (可选，用于反向代理) |

## 二、数据库部署

### 2.1 创建数据库

```sql
-- 创建数据库
CREATE DATABASE bcrm DEFAULT CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;

-- 创建用户（可选，建议生产环境使用独立用户）
CREATE USER 'bcrm'@'%' IDENTIFIED BY 'your_secure_password';
GRANT ALL PRIVILEGES ON bcrm.* TO 'bcrm'@'%';
FLUSH PRIVILEGES;
```

### 2.2 导入初始数据

```bash
# 登录MySQL执行初始化脚本
mysql -u bcrm -p bcrm < back-end/src/main/resources/db/schema.sql
mysql -u bcrm -p bcrm < back-end/src/main/resources/db/data.sql
```

## 三、后端部署

### 3.1 编译打包

```bash
cd code/back-end

# 编译打包（跳过测试）
mvn clean package -DskipTests

# 生成的JAR包位于：target/bcrm-server-1.0.0.jar
```

### 3.2 配置文件

生产环境建议使用外部配置文件 `application-prod.yml`：

```yaml
server:
  port: 8080
  servlet:
    context-path: /api

spring:
  datasource:
    url: jdbc:mysql://localhost:3306/bcrm?useUnicode=true&characterEncoding=utf8&useSSL=false&serverTimezone=Asia/Shanghai
    username: bcrm
    password: your_secure_password
    druid:
      initial-size: 10
      min-idle: 10
      max-active: 50

# JWT配置（生产环境请更换密钥）
jwt:
  secret: your-very-long-and-secure-jwt-secret-key-at-least-64-characters-long-for-production

# 日志配置
logging:
  level:
    com.bcrm: info
    root: warn
  file:
    name: logs/bcrm.log

# 关闭SQL日志
mybatis-plus:
  configuration:
    log-impl: org.apache.ibatis.logging.nologging.NoLoggingImpl
```

### 3.3 启动服务

**前台启动（测试用）：**
```bash
java -jar bcrm-server-1.0.0.jar --spring.profiles.active=prod
```

**后台启动：**
```bash
nohup java -Xms512m -Xmx1024m -jar bcrm-server-1.0.0.jar --spring.profiles.active=prod > logs/console.log 2>&1 &
```

**Systemd服务（推荐）：**

创建服务文件 `/etc/systemd/system/bcrm.service`：

```ini
[Unit]
Description=BCRM Service
After=network.target mysql.service

[Service]
Type=simple
User=bcrm
WorkingDirectory=/opt/bcrm
ExecStart=/usr/bin/java -Xms512m -Xmx1024m -jar /opt/bcrm/bcrm-server-1.0.0.jar --spring.profiles.active=prod
ExecStop=/bin/kill -15 $MAINPID
Restart=on-failure
RestartSec=10

[Install]
WantedBy=multi-user.target
```

```bash
# 启用并启动服务
systemctl daemon-reload
systemctl enable bcrm
systemctl start bcrm

# 查看状态
systemctl status bcrm
```

## 四、前端部署

### 4.1 构建生产版本

```bash
cd code/front-end

# 安装依赖
npm install

# 构建生产版本
npm run build

# 生成的文件位于 dist/ 目录
```

### 4.2 部署静态文件

将 `dist/` 目录内容部署到Web服务器：

**Nginx配置示例：**

```nginx
server {
    listen 80;
    server_name your-domain.com;

    # 前端静态文件
    location / {
        root /var/www/bcrm/dist;
        index index.html;
        try_files $uri $uri/ /index.html;
    }

    # API反向代理
    location /api {
        proxy_pass http://127.0.0.1:8080;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }

    # 文件上传大小限制
    client_max_body_size 50M;
}
```

## 五、安全配置

### 5.1 防火墙配置

```bash
# 开放必要端口
firewall-cmd --permanent --add-port=80/tcp
firewall-cmd --permanent --add-port=443/tcp
firewall-cmd --reload
```

### 5.2 HTTPS配置（推荐）

使用 Let's Encrypt 免费证书：

```bash
# 安装certbot
yum install certbot python3-certbot-nginx

# 申请证书
certbot --nginx -d your-domain.com

# 自动续期
crontab -e
# 添加：0 0 1 * * certbot renew --quiet
```

### 5.3 敏感配置处理

生产环境请务必修改以下配置：

1. **数据库密码**：使用强密码
2. **JWT密钥**：更换为随机生成的长密钥
3. **Druid监控密码**：修改默认账号密码
4. **默认管理员密码**：首次登录后立即修改

## 七、常见问题

### Q1: 后端启动失败，数据库连接错误

检查：
1. MySQL服务是否启动
2. 数据库连接信息是否正确
3. 防火墙是否开放MySQL端口（如远程连接）
4. MySQL用户权限是否正确

### Q2: 前端页面空白

检查：
1. 后端服务是否正常
2. Nginx配置是否正确
3. 浏览器控制台是否有错误

### Q3: API请求跨域

检查：
1. Nginx反向代理配置
2. 后端CORS配置

## 八、备份策略

### 数据库备份

```bash
# 每日自动备份脚本
#!/bin/bash
BACKUP_DIR=/backup/mysql
DATE=$(date +%Y%m%d_%H%M%S)
mysqldump -u bcrm -p'your_password' bcrm > $BACKUP_DIR/bcrm_$DATE.sql
# 保留最近7天备份
find $BACKUP_DIR -name "bcrm_*.sql" -mtime +7 -delete
```

添加到crontab：
```bash
0 2 * * * /opt/scripts/backup_mysql.sh
```
